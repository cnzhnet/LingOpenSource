Wunion.DataAdapter.NetCore 更新日志

1.0.2.0更新内容：
1、更进一步优化了 SpeedDataTable 的性能。
2、修复了一个命令中多次使用相同名称的字段进行 LIKE 查询时，可能发生的数据库参数变量名称重复生成的Bug。
1.0.1.0更新内容：
1、扩展 MySQL 数据库的支持。
2、修复 SpeedDataTable.SerializeJson 方法当某个非字符串类型字段为空值时输出的 json 格式有错误的问题。

 

Wunion.DataAdapter.Kernel 更新日志

2.0.1.0更新内容：
1、扩展 MySQL 数据库的支持。
2、修复 SpeedDataTable.SerializeJson 方法当某个非字符串类型字段为空值时输出的 json 格式有错误的问题。

2.0.0.0重大更新：
1、重构源代码结构，使项目移植到 .NET Core 的工作简化到极至(对应的 .NET Core 支持库为：Wunion.DataAdapter.NetCore)。
2、删除 Wunion.DataAdapter.Kernel.CommandBuilders.BlockDescription 命名空间，代码合并至 Wunion.DataAdapter.Kernel.CommandBuilders
3、删除 Wunion.DataAdapter.Kernel.CommandBuilders.ElementDescription 命名空间，代码合并至 Wunion.DataAdapter.Kernel.CommandBuilders
4、重新设计命令解析器的适配机制，规避多种数据库同时使用时并发解析可能造成的问题。
5、封堵了使用 Like 查询即：exp.Like(...) 所生成的目标 LIKE 子句中包含的 SQL 注入漏洞。

1.0.2.2更新内容：
1、新增 Wunion.DataAdapter.Kernel.DataCollection.SpeedDataTable 类（实现高速绑定的类据表。）
2、DataEngine类新增 ExecuteQuery 方法的重载返回 SpeedDataTable 对象。
使用方法：参见 TestProject 工程

1.0.2.1更新内容：
1、修复 exp.Like(...) 生成的 LIKE 子句包含 SQL 注入漏洞问题。
